# Real State MVC

A full-stack real estate web application built in PHP with a custom MVC architecture. The site lets visitors browse property listings and contact the agency, while a protected admin panel lets staff manage properties and sellers.

Live repo: [Kekekingg/Real_State_MVC](https://realstate-mvc.alwaysdata.net/)

---

## Overview

Real State MVC is a from-scratch MVC implementation (no framework) covering:

- A **public-facing site**: home, about us, property listings, property detail, blog, blog entry, and a contact form.
- A **session-protected admin panel** for full CRUD on properties and sellers, including image upload/processing.
- A **login system** with hashed password verification and route-level access control.
- **Email notifications** via PHPMailer/Mailtrap whenever a visitor submits the contact form.
- A **custom front-end build pipeline** (Sass + vanilla JS) compiled with Gulp.

## Environment Configuration

The app reads environment variables through `vlucas/phpdotenv`. Create a `.env` file inside `includes/` (the same folder as `app.php`, since `Dotenv::createImmutable(__DIR__)` resolves relative to that directory):

```env
DB_HOST=localhost
DB_USERNAME=your_user
DB_PASSWORD=your_password
DB_NAME=your_database
```

> **Note:** the repository's `.gitignore` already excludes `.env` and expects an `.env.example` file, but none is currently committed. See [`docs/TROUBLESHOOTING.md`](docs/TROUBLESHOOTING.md) for details and a suggested template.

## Database Schema

```sql
CREATE TABLE sellers (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    last_name VARCHAR(255) NOT NULL,
    phone VARCHAR(20) NOT NULL
);

CREATE TABLE properties (
    id INT AUTO_INCREMENT PRIMARY KEY,
    title VARCHAR(255) NOT NULL,
    price DECIMAL(10,2) NOT NULL,
    image VARCHAR(255) NOT NULL,
    description TEXT NOT NULL,
    bedrooms INT NOT NULL,
    wc INT NOT NULL,
    parking_space INT NOT NULL,
    created VARCHAR(20) NOT NULL,
    sellers_id INT NOT NULL,
    FOREIGN KEY (sellers_id) REFERENCES sellers(id)
);

CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    email VARCHAR(255) NOT NULL UNIQUE,
    password VARCHAR(255) NOT NULL
);
```

---

To create an admin account, insert a row into `users` with a password generated by PHP's `password_hash()` — the login flow verifies it with `password_verify()`.

## Admin Panel

Once authenticated (`/login`), staff can:

- Create, update, and delete properties (with image upload, resized to 800×600 via Intervention Image).
- Create, update, and delete sellers.
- See success/error notifications after each action (`?result=1|2|3` query flag).

---

## Contact Form

Visitors submit a form (`/contact`) choosing to be reached by **phone** (with a preferred date/time) or **email**. `src/js/app.js` dynamically swaps the relevant fields in, and `PageController::contact` builds an HTML email and sends it through PHPMailer over a Mailtrap SMTP sandbox.

## Known Limitations / Notes for Contributors

- Database queries in `models/ActiveRecord.php` and `models/Admin.php` are built via string concatenation rather than prepared statements; input is escaped with `mysqli::escape_string`, but a migration to parameterized queries is recommended.
- The Mailtrap SMTP credentials are currently hardcoded in `PageController::contact` instead of being pulled from `.env`.
- IDs are recycled by scanning for gaps (`ActiveRecord::getNextId`), which is unconventional for a relational DB — see the architecture doc for reasoning and caveats.
- This project was built as an educational exercise (custom MVC instead of a framework) and can be extended with stronger authentication, request validation, and a proper REST API.
